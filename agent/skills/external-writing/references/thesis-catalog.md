# Thesis Catalog

Use one or two lenses only when they reveal a real evidence-backed tension:

- Feedback loop: what can the system observe about its own output, and what can it correct?
- Cost shift: which old optimum stops making sense after a resource cost changes by an order of magnitude?
- Consensus ceiling: where does generic consensus end and specific context become the differentiator?
- Technical lineage: which earlier generations made the current release possible, and what limitation remains?
- Mechanism beneath the headline: what concrete constraint explains the public narrative?
- Value redistribution: after execution friction falls, where do judgment, taste, and management move?
- Management transfer: which mature organizational practice maps cleanly onto agent reliability?
- Context architecture: what information can each actor see, when, and in what form?

These are private analysis aids. Do not expose lens names, numbers, or catalog language in the final report. A useful thesis is specific, contestable, consequential, and supported by the current evidence pack.
